🔄 Workflow chuyển đổi Keil ↔ STM32CubeIDE

Đọc từ trên xuống để hiểu toàn bộ quy trình.

1

Ban đầu phát triển bằng Keil

📁 Main_Board.ioc
Core/
Drivers/
Main_Board.uvprojx
➡
Firmware được build & debug bằng Keil

Dự án được tạo bằng STM32CubeMX và chọn Toolchain = MDK-ARM (Keil). CubeMX sinh ra file project của Keil (.uvprojx, .uvoptx) cùng với mã nguồn (Core, Drivers) và file cấu hình .ioc.

2

Sau đó chuyển sang STM32CubeIDE

Mở Main_Board.ioc
➡
Project Manager
➡
Toolchain = STM32CubeIDE
➡
Generate Code

CubeMX sẽ tạo thêm các file .project, .cproject và thư mục .settings. Đây là các file mà STM32CubeIDE sử dụng để nhận diện và build project.

3

Điều gì được giữ nguyên?

Source firmware không thay đổi, chỉ project của IDE được tạo lại.
  • ✅ Core giữ nguyên.
  • ✅ Drivers giữ nguyên.
  • ✅ Middlewares giữ nguyên.
  • ✅ Source code giữ nguyên.
  • 🔁 Chỉ các file project của IDE (.uvprojx hoặc .project/.cproject) được sinh lại.
4

Nếu tự thêm source code thì sao?

motor.c / motor.h
➡
Generate Code
➡
USER CODE BEGIN...END

Bạn có thể tự tạo các file như motor.c và motor.h. Khi Generate Code, CubeMX sẽ giữ nguyên các file người dùng và không ghi đè phần mã nằm trong vùng USER CODE BEGIN và USER CODE END.

5

Khi nào cần Generate Code?

Đổi Toolchain
➡
Generate Code
➡
IDE mới mở được project

Mỗi lần chuyển giữa Keil ↔ STM32CubeIDE, hãy mở lại file .ioc, đổi Toolchain và Generate Code. CubeMX sẽ sinh đúng project cho IDE mới trong khi toàn bộ firmware vẫn được giữ nguyên.
